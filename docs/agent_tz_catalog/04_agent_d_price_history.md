# ТЗ для Агента D: история изменений цен и аудит

## Цель
Получить прозрачный журнал изменений цен для разборов ошибок и контроля процесса.

## Объем работ
1. Добавить миграцию:
   - `migrations/008_product_price_history.sql`.
2. Создать таблицу `product_price_history`:
   - `id`, `product_id`, `changed_by`, `reason`,
   - `old_values_json`, `new_values_json`,
   - `created_at`.
3. В `products.php` при каждом изменении цен:
   - сохранять снимок старых и новых значений;
   - писать причину (минимум: "manual_edit"/"bulk_update"/"sync_fix").
4. На странице каталога добавить:
   - кнопка "История" по товару;
   - всплывающее окно/блок с последними 10 изменениями.

## Ограничения
- Не внедрять бизнес-логику ценообразования (агент B).
- Не внедрять sync-очередь (агент C).
- Не делать крупный UI-редизайн (агент A).

## Технические требования
- Изменения должны быть легковесными: запись истории не должна тормозить массовые обновления.
- JSON хранить в `UTF-8`, без потери точности decimal-полей.
- Падение истории не должно отменять сохранение цены (логировать и продолжать).

## Acceptance Criteria
- Каждое изменение цен фиксируется в истории.
- По товару можно быстро понять, кто/когда/что поменял.
- История корректно отображается в интерфейсе без падений.
