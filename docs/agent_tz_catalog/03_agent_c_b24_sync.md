# ТЗ для Агента C: надежный sync каталога цен с B24

## Цель
Сделать синхронизацию цен в B24 управляемой и прозрачной: видно статус по товару, можно повторить точечно.

## Объем работ
1. В `products.php` внедрить статусы sync по товарам:
   - `pending`, `sent`, `error`;
   - текст последней ошибки;
   - время последней попытки.
2. Сделать операции:
   - sync одной строки;
   - массовый sync выбранных строк;
   - retry только по `error`.
3. Сохранить подход "локальное сохранение не падает из-за B24":
   - сначала local DB, затем sync;
   - при провале sync сохранять ошибку и давать retry.
4. Добавить rate-friendly обработку:
   - батч по 20-50;
   - небольшой delay между запросами;
   - аккуратное логирование ошибок API.

## Ограничения
- Не менять core валидацию цен (агент B).
- Не менять историю аудита (агент D).
- Не переделывать глобально UI (агент A), только нужные элементы статуса/кнопок.

## Технические требования
- Переиспользовать `syncProductPriceToB24()` как базу, расширить безопасно.
- При необходимости добавить настройки батча в `app_settings`.
- Не трогать `b24_sales.php`, `stock_operations.php`, `sync_monitor.php` в этом блоке.

## Acceptance Criteria
- По каждому товару в таблице виден последний результат sync.
- Retry по ошибкам работает точечно без полного "прогона всех".
- Массовый sync не блокирует работу страницы и корректно показывает итог.
