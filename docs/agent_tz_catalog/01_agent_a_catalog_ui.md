# ТЗ для Агента A: UI/UX каталога и inline-редактирование

## Цель
Сделать страницу `products.php` удобной для ежедневной массовой правки цен без перехода в форму "один товар за раз".

## Объем работ
1. Перевести список товаров в удобную таблицу управления:
   - sticky header;
   - компактные колонки;
   - пагинация (минимум 50/100/200 строк);
   - поиск по названию.
2. Добавить фильтры:
   - `catalog_id`;
   - бренд (по префиксу имени);
   - наличие `b24_product_id`;
   - "есть пустые цены".
3. Добавить inline-редактирование в строке:
   - `roll_length`, `purchase_price`, `delivery_price`, `price_per_meter`,
   - `price_1_4`, `price_5_9`, `price_10_19`, `price_20_plus`.
4. Добавить кнопки в строке:
   - `Сохранить`;
   - `Отменить`;
   - `Синк B24` (вызов существующей server-side логики, без редизайна backend).
5. Добавить режим массового выделения строк (checkbox), пока без бизнес-операций (их делает агент B/C).

## Ограничения
- Не менять расчетные правила цен (это зона агента B).
- Не менять логику sync/retry (это зона агента C).
- Не внедрять историю изменений (это зона агента D).

## Технические требования
- Файл: `products.php` + стили в `assets/style.css`.
- Сохранить совместимость с текущими формами и роутингом.
- Если добавляешь JS, использовать vanilla JS.

## Acceptance Criteria
- Пользователь может отфильтровать и отредактировать 20+ товаров подряд без открытия отдельных карточек.
- Ошибки валидации отображаются рядом со строкой, не ломая страницу.
- Время реакции интерфейса комфортное на 1000+ товарах (с пагинацией).
