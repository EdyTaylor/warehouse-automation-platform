# Настройка интеграции с Битрикс24

## Текущий статус интеграции

✅ **Уже работает:**
- Импорт товаров из Б24 в приложение
- Синхронизация остатков со склада в Б24
- Продажи из Б24 с автоматическим списанием
- Двусторонняя синхронизация цен

✅ **Новые возможности:**
- Удобная панель кладовщика (`dashboard.php`)
- Обработка заказов из Б24 (`process_order.php`)
- Автоматические вебхуки для мгновенной синхронизации

## Настройка вебхуков в Битрикс24

### 1. Создайте входящий вебхук

1. Зайдите в Битрикс24: https://llumar.bitrix24.kz
2. Перейдите: `Приложения` → `Разработчикам` → `Вебхуки`
3. Нажмите `Добавить вебхук`
4. Выберите права доступа:
   - `crm` (доступ к CRM)
   - `catalog` (доступ к каталогу)

### 2. Настройте outbound вебхуки

Добавьте следующие URL в настройках Б24 для автоматической отправки событий:

**Базовый URL вашего приложения:** `https://your-domain.com/api/webhook.php`

#### События для настройки:

1. **Новая сделка:**
   - Событие: `ONCRMDEALADD`
   - Обработчик: `https://your-domain.com/api/webhook.php`

2. **Обновление сделки:**
   - Событие: `ONCRMDEALUPDATE` 
   - Обработчик: `https://your-domain.com/api/webhook.php`

3. **Новый товар:**
   - Событие: `ONCRMPRODUCTADD`
   - Обработчик: `https://your-domain.com/api/webhook.php`

4. **Обновление товара:**
   - Событие: `ONCRMPRODUCTUPDATE`
   - Обработчик: `https://your-domain.com/api/webhook.php`

## Настройка полей в Битрикс24

### 1. Поле для остатков на складе

Создайте пользовательское поле для товаров:
- **Название:** `Остаток на складе (м)`
- **Код:** `UF_CRM_STOCK_M`
- **Тип:** Число

### 2. Настройте цены

Убедитесь что у товаров есть поле `PRICE` для цены за единицу.

## Использование системы

### Для кладовщика:

1. **Главная страница:** `dashboard.php`
   - Показывает новые заказы из Б24
   - Отображает остатки на складе
   - Быстрые действия по добавлению товара

2. **Обработка заказов:** `process_order.php`
   - Удобный интерфейс для списания товаров
   - Автоматический расчет раскроя
   - Интеграция с движениями склада

3. **Добавление товара:** `add_stock.php`
   - Быстрое добавление рулонов на склад
   - Автоматическое логирование движений

### Для менеджера:

1. **Продажи в Б24:**
   - Создайте сделку с товарами
   - Заказ автоматически появится у кладовщика
   - Кладовщик обработает и спишет товар

2. **Синхронизация цен:**
   - Измените цену в приложении → обновится в Б24
   - Измените цену в Б24 → обновится в приложении

## API endpoints

### Синхронизация остатков:
```
GET /api/bitrix/sync_stock.php?push=1
```

### Синхронизация цен:
```
GET /api/sync_prices.php?action=to_b24    # Из приложения в Б24
GET /api/sync_prices.php?action=to_app     # Из Б24 в приложение
```

### Импорт товаров:
```
GET /api/bitrix/import_products.php
```

## Автоматизация

### 1. Настройте cron для синхронизации:

```bash
# Каждые 5 минут синхронизировать остатки
*/5 * * * * curl https://your-domain.com/api/bitrix/sync_stock.php?push=1

# Каждый час синхронизировать цены
0 * * * * curl https://your-domain.com/api/sync_prices.php?action=to_app
```

### 2. Вебхуки обеспечивают мгновенную синхронизацию:
- Новая сделка в Б24 → мгновенно появляется у кладовщика
- Изменение цены → синхронизация в обе стороны
- Добавление товара → автоматический импорт

## Решение проблем

### Проверка вебхуков:
```sql
SELECT * FROM webhook_log ORDER BY created_at DESC LIMIT 10;
```

### Проверка движений склада:
```sql
SELECT * FROM stock_movements ORDER BY created_at DESC LIMIT 10;
```

### Проверка статусов синхронизации:
```sql
SELECT * FROM stock_movements WHERE bitrix_status = 'error';
```

## Безопасность

1. Ограничьте доступ к API по IP
2. Используйте HTTPS
3. Регулярно проверяйте логи вебхуков
4. Создайте резервные копии БД

## Поддержка

При возникновении проблем:
1. Проверьте логи вебхуков в таблице `webhook_log`
2. Проверьте настройки вебхуков в Б24
3. Убедитесь что все миграции БД применены
4. Проверьте права доступа в Б24
